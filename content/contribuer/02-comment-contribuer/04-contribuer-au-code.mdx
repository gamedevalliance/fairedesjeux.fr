Cette dernière section explique comment contribuer à l'aspect technique du site. Si votre objectif est uniquement d'écrire du contenu, vous pouvez passer votre chemin, car vous savez déjà tout ce qu'il vous faut !

### Structure du repo

Pour vous aider à vous familiariser avec le repo, voici sa structure :

- `tailwind.config.cjs` : Configuration de Tailwind
- `astro.config.mjs` : Configuration d'Astro
- `src/` : Code source du projet
    - `components/` : Composants réutilisables dans les pages web, tels que les bulles de personnage, les vidéos et bien d'autres. Il est très simple de s'inspirer des composants existants pour créer un nouveau composant !
    - `layouts/` : Composants généraux structurant les pages.
    - `pages/` : Composants représentant des pages.
- `content/` : Le contenu du site (fichiers Markdown, images et vidéos)

Pour en savoir plus sur la structure d'un projet Astro, référez-vous à la [documentation officielle](https://docs.astro.build/core-concepts/project-structure/).

### Conventions de style

Pour respecter les conventions du repo, nous utilisons [Prettier](https://prettier.io/), disponible sous la forme d'extension pour la plupart des éditeurs de code ou bien par installation manuelle.

### Créer une pull request

Pour que votre pull request soit acceptée, nous vous demandons bien sûr de suivre les conventions de style, mais aussi de ne pas inclure de fichier inutile au repo : ajoutez des règles à `.gitignore` si nécessaire. Si vous ajoutez une dépendance, précisez pourquoi elle est nécessaire à moins que ce ne soit évident. Merci beaucoup pour l'intérêt que vous portez au projet !
